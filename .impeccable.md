## Design Context

### Users
Internal Automattic employees. They live in P2s — browsing, commenting, writing posts across the entire network of Automattic P2 sites. This is a daily-driver tool for people who are already fast and opinionated about their tools. They'll judge it against the best productivity software they use, not against WordPress admin.

### Brand Personality
**Dense, confident, fast.** Not playful-cute, not enterprise-gray. The interface should feel like it respects your time and assumes you know what you're doing. Information-rich without feeling cluttered. Responsive to interaction without being showy about it.

### Aesthetic Direction
- **Theme:** System-aware dark/light mode with a user override preference. Dark is the lead context (many Automatticians work long hours across time zones).
- **Anti-references:** Not Slack (too chatty, too much chrome). Not Linear (too sparse, too precious about whitespace). Cortex should be denser than both — closer to a power-user tool that earns its complexity.
- **Visual tone:** Modern, fun, fast. The kind of interface where everything feels one click away. Density is a feature, not a compromise.
- **Color:** WordPress brand blue (#3858e9) as accent for now. Will eventually align with the Automattic (a8c) brand directly.
- **Design system:** Built on `@wordpress/ui`, `@wordpress/theme`, and `@wordpress/icons`. ThemeProvider handles dark/light token generation from seed colors. Components come from the WPDS; custom styling fills the gaps.

### Typography
- **Display:** Bricolage Grotesque 800 (Google Fonts, variable with optical sizing 12–96). Used for page titles, the wordmark, and any hero-level text. Tight tracking (-0.03em), line-height 1.
- **Body / UI:** WPDS system font stack via `--wpds-typography-font-family-body` (-apple-system, system-ui, Segoe UI, Roboto, etc.). All WPDS `Text` component variants for UI text.
- **Scale:** Display uses fluid sizing via `clamp()`. App UI uses the fixed WPDS type scale (body-sm through heading-2xl).

### Motion
- **Easing:** `cubic-bezier(0.16, 1, 0.3, 1)` (ease-out-expo) for all entrance animations.
- **Stagger:** 80ms between sequential elements.
- **Duration:** 0.7s for entrances, 0.15s for hover/interactive transitions.
- **Reduced motion:** All animations gate on `prefers-reduced-motion`.

### Spacing
- Use WPDS dimension tokens (`--wpds-dimension-gap-*`, `--wpds-dimension-padding-*`) for all spacing.
- Use `clamp()` for page-level padding that scales with viewport.
- Vary spacing for hierarchy — tighter grouping within related elements, generous separation between sections.

### Design Principles
1. **Density is respect.** Show more, scroll less. Every pixel should earn its place, but never feel cramped. The difference is typographic hierarchy and deliberate spacing rhythm.
2. **Speed is the feature.** Interactions should feel instant. Optimistic UI, minimal loading states, transitions that finish before you notice them. If something takes time, be honest about it — no fake spinners on fast operations.
3. **Confidence over cleverness.** Strong defaults, not endless options. The interface makes decisions so the user doesn't have to. When there's a choice, pick the smart default and let power users override.
4. **System-native feel.** Respect the OS. Dark/light follows the system. Keyboard shortcuts feel natural. Scrolling is native. Nothing fights the platform.
5. **WordPress DNA, not WordPress admin.** Use the WPDS as foundation but push beyond the wp-admin aesthetic. This is a modern client app that happens to speak WordPress, not a WordPress plugin.
